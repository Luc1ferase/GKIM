## 2. Repository active-path projection

(Scope clarification: `DefaultCompanionTurnRepository` already tracks `variantGroups: Map<variantGroupId, VariantGroupState>` per conversation, where `VariantGroupState` carries `siblingMessageIds + activeIndex`, and exposes `selectVariant(turnId, variantIndex)` for mutation. The §3.1 chevron rendering reads `ChatMessage.companionTurnMeta.siblingCount` + `siblingActiveIndex` — fields the repository's `resolveActivePath` projection does NOT currently populate, so chevrons never render in production today. This section closes that projection gap and exposes the variantGroupId-keyed mutation path the §3.1 `onSelectVariantAt(variantGroupId, newIndex)` chevron callback needs.)

- [x] 2.1 Extend `DefaultCompanionTurnRepository.resolveActivePath` so each rendered `ChatMessage` whose `companionTurnMeta != null` carries `siblingCount = variantGroups[meta.variantGroupId].siblingMessageIds.size` and `siblingActiveIndex = variantGroups[meta.variantGroupId].activeIndex`. Single-sibling groups MUST project `siblingCount = 1` (so the §3.1 chevron suppression rule applies). Verification: `.\gradlew.bat --no-daemon :app:testDebugUnitTest --tests com.gkim.im.android.data.repository.CompanionTurnRepositorySiblingProjectionTest` covers single-sibling / two-sibling-after-regenerate / three-sibling-after-edit-and-regenerate cases.

- [ ] 2.2 Add `CompanionTurnRepository.selectVariantByGroup(conversationId, variantGroupId, newIndex)` (interface method + Default implementation) that mutates the matching `variantGroups` entry's `activeIndex` directly (the §3.1 `onSelectVariantAt` callback's payload is `(variantGroupId, newIndex)`, not the legacy `(turnId, variantIndex)`). The mutation MUST be idempotent (calling with the current index is a no-op) and MUST clamp to `[0, siblingMessageIds.size - 1]`. The existing `selectVariant(turnId, ...)` method stays for back-compat. Verification: `.\gradlew.bat --no-daemon :app:testDebugUnitTest --tests com.gkim.im.android.data.repository.CompanionTurnRepositorySelectVariantByGroupTest` covers the mutation, idempotency, clamping, and unknown-group-no-op branches.
